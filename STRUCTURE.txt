KEY FEATURES
============

1. AI Provider System
   - Primary: LM Studio (local models)
   - Fallback: OpenAI, Anthropic, Gemini (optional)
   - Unified interface with automatic failover

2. Multi-Agent Pipeline
   - PlannerAgent: Breaks down user requests
   - CodeGeneratorAgent: Creates files
   - Extensible for more agents

3. Virtual File System
   - Version control with history
   - Diff tracking
   - MongoDB persistence
   - In-memory cache

4. Docker Sandbox
   - Isolated execution
   - Resource limits (CPU, memory)
   - Network isolation
   - Live log streaming

5. Monaco Editor
   - Syntax highlighting
   - IntelliSense
   - Multi-file editing
   - VS Code experience

6. Real-time Updates
   - WebSocket streaming
   - Live logs
   - Progress updates

API ENDPOINTS
=============

Authentication:
  POST   /api/auth/register     - Register new user
  POST   /api/auth/login        - Login
  GET    /api/auth/me           - Get current user

Projects:
  GET    /api/projects          - List all projects
  GET    /api/projects/:id      - Get project
  POST   /api/projects          - Create project
  DELETE /api/projects/:id      - Delete project

Files:
  GET    /api/projects/:id/files     - List files
  GET    /api/projects/:id/files/*   - Get file
  POST   /api/projects/:id/files     - Create file
  PUT    /api/projects/:id/files     - Update file
  DELETE /api/projects/:id/files     - Delete file

Build:
  WS     /api/build/:projectId       - WebSocket build

Health:
  GET    /health                     - Health check

  - executions     # Agent execution logs

DEPLOYMENT
==========

Quick Install:
  1. Copy to /tmp/emergent-clone
  2. sudo ./install.sh
  3. Enter LAN IP and LM Studio URL
  4. Access http://YOUR_IP:3000

Services:
  - emergent-backend  (port 4000)
  - emergent-frontend (port 3000)
  - mongod            (port 27017)
  - redis-server      (port 6379)
  - nginx             (port 80)

DEVELOPMENT
===========

Backend:
  cd backend
  yarn install
  yarn dev          # Development mode
  yarn build        # Production build
  yarn test         # Run tests

Frontend:
  cd frontend
  yarn install
  yarn dev          # Development mode
  yarn build        # Production build
  yarn start        # Production server

Docker Compose:
  docker-compose up -d     # Start all services
  docker-compose down      # Stop all services
  docker-compose logs -f   # View logs
